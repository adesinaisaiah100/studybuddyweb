# Dashboard Redesign Ideas

These are three high-impact design ideas to revamp the Course Dashboard, enhancing both its utility and visual aesthetics.

## 1. Contextual Quick-Action Menus
Instead of forcing the user to click into the dedicated Course page just to delete or edit a typo, we can add a sleek 3-dot `...` icon in the top-right corner over the `BookOpen` icon. Dropping down a tiny floating menu directly over the glass card ("Edit", "Upload Notes", "Delete") reduces friction. This makes frequent actions much more accessible.

## 2. Introduce Progress / Timeline Activity
A dashboard shouldn't just be a static list of courses. Consider upgrading the cards to display active tracking:
* Have a tiny "Next Class:" snippet highlighting only the single upcoming schedule slot for the current day. 
* Add a progress bar on the card summarizing "Weekly tasks done" or "Syllabus progress", instantly making the dashboard feel like a lively, utility-driven tracker. This turns the dashboard into a daily hub rather than just a directory.

## 3. Toggleable Views (List vs Grid) + Filter Bar
As users upload more timetables over their years in college, they will be flooded with course cards. 
We can add a compact "List View" toggle next to a visual Search Bar at the top. Grid view stays for aesthetic browsing, list view for serious syllabus management. We can also add quick sort pills beneath the greeting: `[Sort by Code] [Sort by Next Class]`.
